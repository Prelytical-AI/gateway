Phase 1 Only · NSAW Sprint

Metadata in · Human review · Cloud processing

The default first engagement for NetSuite Analytics Warehouse clients: no client VM, no row-level data, frontier model on Prelytical SaaS — with explicit human approval before anything is processed.

In one sentence: Client exports NSAW metadata → humans on both sides review → Prelytical cloud generates the brief with an approved frontier model → consultant delivers after final review.

This document describes only Phase 1. It does not cover private GPU inference, live ADW connections, or the on-prem SQL gateway. Those are later phases if the client needs them.

Step-by-step flow

Every step is designed to be explainable in a security conversation.

1
Client

Create domain & business context

CFO goals, pain points, metrics, audience — entered in Prelytical or collected in a kickoff worksheet. No warehouse access needed yet.

2
Client

Export NSAW metadata package

DBA or implementation partner pulls catalog from ADW / NSAW semantic model: table names, columns, types, keys, relationships, row counts. Mode A — metadata only.

3
Human gate

Client reviews package before upload

Client opens JSON/CSV export and confirms: no sample rows, no PII values, no transaction detail. They sign off or redact before sending.

4
Prelytical cloud

Upload source to platform

Package ingested as a source artifact. Stored in Prelytical workspace — not merged into a shared data lake of client transactions.

5
Human gate

Prelytical confirms run policy

Consultant verifies: metadata_only, row_level_data=false, approved model, correct domain context. Generation does not start until this is explicit.

6
Prelytical cloud

Generate in cloud with frontier model

Shape map → ranked opportunities → executive brief HTML. All inference on Prelytical infrastructure. Audit record captures exactly what metadata fields were sent.

7
Human gate

Consultant final review & delivery

Founder/consultant edits for accuracy, tone, and client context. Client receives polished HTML/PDF — not raw model output.

Three human checkpoints: (1) client approves metadata package, (2) Prelytical confirms policy before generation, (3) consultant approves before client delivery. AI never runs on unreviewed row-level data because row-level data is never in scope.

What crosses the line — and what does not

Included (with review)

  • Schema and table names
  • Column names and data types
  • Primary / foreign keys
  • Row counts (aggregate)
  • Business context (client-authored)
  • Finished brief HTML (outbound deliverable)

Excluded — always in Phase 1

  • NetSuite transaction rows
  • Invoice / payment / employee detail
  • Sample values from sensitive columns
  • Live query results from ADW
  • Credentials to NetSuite or ADW

Run policy (locked for Phase 1)

Every brief run uses this policy. No exceptions in the sprint.

Sprint deliverables (~2 weeks)

Artifact Description
Data shape map Entities, relationships, coverage — what NSAW can support analytically
Opportunity backlog Ranked analysis ideas with readiness, value, and privacy notes
Executive brief Standalone HTML — readiness scores, opportunities, risks, next steps
Deep-dive plan What to validate next if client moves to row-level analysis later
Audit summary What metadata was used, which model, confirmation no rows were included

Explicitly not Phase 1

If the client asks for these, that is a different phase or a different product path.

Request Phase 1 answer
Install gateway on our server Not needed — metadata package upload instead
Connect live to our ADW Phase 2 — still metadata-only by default
Run AI inside our network Phase 3 — private GPU in their tenancy
Ask questions against live data Out of scope — planning brief, not BI chat
Use our M365 Copilot Not integratable — Prelytical uses approved API model

Not a “NetSuite gateway” install. Phase 1 is export → review → upload. The on-prem SQL gateway in the repo is a separate path for SQL Server clients.
