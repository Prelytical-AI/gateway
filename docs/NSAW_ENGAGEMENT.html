Recommended architecture — three phases

Start with Phase 1 for every NSAW prospect. Phase 3 is an enterprise upsell, not the front door.

Phase 1 · Default Recommended start

Metadata-only sprint → Prelytical SaaS

2-week engagement · no client VM · inspect-before-upload metadata package

┌──────────────────────────────────────┐ │ CLIENT (minimal) │ │ NSAW / ADW (Oracle cloud) │ │ Prelytical Connector → metadata pkg │ │ Client inspects package before upload │ └──────────────────┬───────────────────┘ │ HTTPS │ metadata + business context ▼ ┌──────────────────────────────────────┐ │ PRELYTICAL SAAS │ │ Domains · sources · Brief Studio │ │ Approved cloud model (fast) │ │ Audit: what metadata was used │ └──────────────────────────────────────┘

Pros

  • Fastest sales cycle — no GPU, no Windows VM
  • Aligns with “no row-level data” promise
  • Client can review package before upload
  • Best quality for brief prose (frontier model OK)
  • Fits how NSAW teams already think (cloud analytics)

Trade-offs

  • Schema metadata crosses to Prelytical cloud
  • Requires security sign-off on metadata egress
  • No live SQL Ask on ADW in this phase
  • Connector/package flow still being productized

Connector operating modes

Default to Mode A for every first engagement. Mode B only with explicit security approval.

Mode Included Excluded Use when
A · Metadata-only Default Schemas, tables, columns, types, keys, indexes, row counts, object names, relationships Transaction rows, sample values, PII Every sprint · security-first conversations
B · Metadata + profiling Mode A + min/max dates, null %, cardinality, enum hints on non-sensitive fields Raw detail rows Client approves aggregate stats · richer brief quality

What not to lead with for NSAW

Common mis-positioning that slows deals or fails security review.

Windows SQL Gateway install

NSAW is Oracle ADW — not SQL Server. Gateway POC is for a different entry path.

Ollama on the warehouse box

ADW is Oracle-managed cloud. GPU belongs in client tenancy if needed at all.

M365 Copilot as backend

Not a pluggable API. Azure OpenAI is the realistic tenant-bound alternative.

Row-level Ask in sprint

Wrong privacy story for first close. Metadata-first briefs are the product.

NSAW sprint readiness checklist

  • Confirm client has NSAW provisioned (ADW + OAC)
  • Identify NSAW admin / DBA for metadata export
  • Security approves metadata-only package upload
  • Collect business context (CFO goals, pain points, metrics)
  • Choose Mode A or B and document in audit policy
  • Run connector export — client inspects package
  • Generate brief in Prelytical workspace
  • Consultant / founder human review before delivery
  • Export HTML brief + opportunity backlog
  • Discuss Phase 2 only if sprint succeeds
