The qualifying question (first call)

Ask this before scoping architecture or pricing.

“Are you comfortable with a metadata-only export of your NSAW model — table and column names, keys, and relationships, no transaction rows — being reviewed by your team and then processed in Prelytical’s secure environment to produce an executive brief?”

If yes + OK with cloud AI Type A — default sprint. Send package sample + DPA if needed.
If yes + no OpenAI Type B — Prelytical private GPU. Emphasize subprocessor list.
If “need to ask InfoSec” Normal. Type A or B after review. Send one-pager early.
If schema can’t leave Type C or pause. Do not force Type A/B.

Who says yes, hesitates, or says no

Use this to set expectations on cycle length and architecture — not to stereotype individuals.

Usually yes (~60–70%)

Phase 1 Type A or B with light review

  • Standard NSAW + OAC finance/ops teams
  • CFO / VP Finance as executive sponsor
  • Already cloud-comfortable (NetSuite, Oracle)
  • $10K advisory / sprint buyers
  • “No OpenAI” but OK with Prelytical → Type B

Hesitates (~15–25%)

Type B + 1–3 week security/legal cycle

  • InfoSec / IT vendor review required
  • PE-backed formal procurement
  • Regulated adjacency (health, lending, gov contractor)
  • “Nothing external” culture but negotiable
  • Needs DPA, retention policy, sample package

Often no to Type A/B (~10–15%)

Type C, later, or not a fit

  • Hard no data egress policies
  • Schema itself treated as confidential
  • Air-gapped / on-prem only mandates
  • Expect processing only inside ADW
  • Won’t use any vendor cloud for analytics inputs

Buyer personas & likely choice

Map the room on discovery — sponsor, blocker, and technical gatekeeper often disagree.

📊

Executive sponsor

CFO · VP Finance · COO

Wants cash, margin, close speed, ranked opportunities — not architecture debates.

Usually Type A
  • “Just get me the brief.”
  • Often already bought NSAW — cloud is fine
  • Will push past InfoSec if value is clear
⚙️

NetSuite / NSAW admin

ERP admin · BI lead · Implementation partner

Owns the model, can export metadata, cares about accuracy of table/column context.

Type A or B
  • Comfortable with metadata exports
  • May prefer Type B if org bans external AI
  • Key ally for package quality
🔒

InfoSec / compliance

CISO delegate · IT security · Internal audit

Cares about subprocessors, retention, what’s in the package, audit trail.

Often pushes Type B
  • May block Type A (OpenAI) outright
  • Type B removes frontier subprocessor
  • Still may require DPA + sample package review
🖥️

IT / infrastructure

IT director · Cloud architect

Worries about installs, VPNs, new VMs, support burden.

Prefers Type A or B
  • Type A/B = no client VM for sprint
  • Type C = “another box to patch” — resistance common
  • Ally for “no install” narrative
📁

PE / portfolio ops

Operating partner · Value creation · M&A diligence

Wants repeatable playbook across portcos; formal vendor process.

Type B after review
  • Standardize on metadata-only + one vendor
  • Often prohibits ad-hoc OpenAI use → Type B
  • Budget for legal review across deals
🏛️

Regulated / conservative legal

General counsel · Privacy officer

Column names may imply PHI/PII; treats any external processing as risk.

Type C or delay
  • May reject Type A and B entirely
  • Ask for redacted / scoped metadata subset
  • Human review non-negotiable
